**User:** When does a wrapper need its own `withBreakpoints` wrap, and which `website-2025/src/nice/*` wrappers actually need it?

**Claude (16:30, react-styles):** A wrapper needs its own `withBreakpoints` only if its body reads a viewport-resolved prop value before delegating to the inner core. Building, forwarding, or ignoring the `breakpoints` array as data does not require the wrap — only consuming a resolved value at the wrapper's render scope does. Verified the three website-2025 wrappers (`nice/Tile`, `nice/Flex`, `nice/Typography`) by reading each: every one is `{ ...defaults, ...props }` pass-through with no body-level inspection. Each inner core (`nice-react-tile`, `nice-react-flex`, `nice-react-typography`) is already wrapped at its component-folder `index.ts`, so consumer `breakpoints={[...]}` resolves at the inner layer regardless. Removing the outer wraps is purely simplification — no behavioral change.
